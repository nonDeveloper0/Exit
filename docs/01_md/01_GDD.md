# EXIT 2026 GDD

## 목표
현대판 모세 이야기를 크라임씬 형태로 체험한다.
참가자들은 모세 이야기를 체험하고 있다는 사실을 모른 채 수사에 참여한다.

## 규모
- 인원: 80명
- 조: 숫자 조 번호로 구분 (1조, 2조, ... 확장 가능)
- 시간: 120분

## 팀 구조
- 같은 조 번호를 입력한 기기끼리 증거 수집함 실시간 공유
- 조 번호는 숫자로 자유 입력 — 조 수 제한 없음

## 핵심 반전
최종 엔딩에서 용의자 C의 정체가 모세임이 밝혀진다.

---

## 게임 흐름

```
입장 → 조 번호(숫자) + 조장 이름 입력
    ↓
사건 브리핑 (메인 화면)
    ↓
현장 이동 → QR 스캔 → 증거 수집 (같은 조와 실시간 공유)
    ↓
증거 분석 (증거함) + 용의자 검토 (용의자 화면)
    ↓
조별 최종 투표 (범인 선택 → Google Form 전송)
    ↓
엔딩 공개 (반전: C = 모세)
```

## QR 워크플로우

```
현장 QR 스캔 → /qr/[slug] 접속
    ↓
해당 QR의 증거 1~2개 공개
    ↓
수집 버튼 클릭 → Supabase에 저장
    ↓
같은 조 다른 기기에 실시간 반영
```

## QR 구조
- 장소 4개 (살해 현장, CCTV 관제실, 주차장, 창고)
- QR 코드 6개 — 각 QR에 1~2개 증거 연결
- 장소명은 `data.ts`의 `LOCATIONS` 상수에서 관리

## 기술 특이사항
- 증거 수집 상태는 Supabase에 저장 → 같은 조 기기 간 실시간 공유
- 투표/팀 정보는 각 기기 localStorage에 저장
- 투표 결과만 Google Form으로 전송
