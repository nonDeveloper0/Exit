# UI SPEC

## 페이지 목록

| 경로 | 페이지명 | 설명 |
|------|----------|------|
| `/` | 랜딩 | 조 번호(숫자) + 조장 이름 입력 → 입장하기 (애니메이션 배경) |
| `/home` | 수사본부 (메인) | 사건 브리핑, 내 조 증거 수집 진행률, QR 수집 현황 |
| `/qr/[slug]` | QR 증거 수집 | 해당 QR의 증거 1~2개 공개 및 수집 |
| `/evidence` | 증거함 | 수집한 증거 전체 목록 |
| `/suspects` | 용의자 | A, B, C, D, E 카드 (동기, 설명) |
| `/ranking` | 수사 현황 | 전체 조 증거 수집 실시간 랭킹 |
| `/vote` | 최종 투표 | 용의자 선택 → Google Form 전송 (팀 정보는 랜딩에서 저장) |
| `/ending` | 엔딩 | 반전 공개 (C = 모세) |
| `/reset` | 관리자 초기화 | 조별 개별 초기화 또는 전체 일괄 초기화 |

---

## 각 페이지 상세

### 랜딩 (`/`)
- 애니메이션 배경 (blob + grid overlay)
- 2026 SUMMER CAMP / EXIT / SEASON 1 타이틀
- 조 번호 숫자 입력 (양의 정수, 같은 번호 입력 시 증거 공유)
- 조장 이름 입력
- 입장하기 버튼 → `/home` 이동 + localStorage 저장
- 재방문 시 기존 정보 prefill

### 수사본부 (`/home`)
- 사건명, 브리핑 텍스트
- 내 조 증거 수집 진행률 표시 (Supabase 기반, 조 공유)
- QR 수집 현황: QR1~QR6 각각 미수집/일부수집/완료 상태 표시

### QR 증거 수집 (`/qr/[id]`)
- 장소명 헤더 (LOCATIONS 상수에서 가져옴)
- 연결된 증거 1~2개 공개
- 증거별 수집 버튼 (수집 완료 시 체크 표시, 조 전체에 실시간 반영)
- 잠긴 증거: 비밀번호 입력 후 수집 가능

### 증거함 (`/evidence`)
- 수집한 증거만 표시 (조 공유 상태 실시간 반영)
- 미수집 증거는 잠김 상태로 표시

### 용의자 (`/suspects`)
- A, B, C, D, E 카드 형태
- 이름, 직책, 동기, 설명 표시
- 동기는 motiveRevealIds 증거 수집 시 공개

### 최종 투표 (`/vote`)
- 용의자 선택 (A / B / C / D / E)
- 상단에 조 번호 · 조장 이름 표시 (랜딩에서 저장된 값)
- 제출 → Google Form으로 전송
- 제출 후 선택 내용 표시 (최대 2회 재제출 가능)

### 수사 현황 (`/ranking`)
- 전체 조 증거 수집 실시간 랭킹 (수집 개수 내림차순)
- 1~3위 색상 구분 (amber / silver / bronze)
- 내 조 amber 강조 표시
- 실시간 업데이트 (Supabase Realtime)

### 엔딩 (`/ending`)
- 반전 공개: C = 모세
- 스토리 마무리 텍스트

### 관리자 초기화 (`/reset`)
- Supabase에 기록 있는 조 목록 자동 표시 (수집 개수 포함)
- 조별 개별 초기화 버튼
- 전체 조 일괄 초기화 버튼
- 현재 기기 조 reset 시 localStorage(투표 기록)도 함께 삭제
