# CLAUDE.md

LLM의 흔한 코딩 실수를 줄이기 위한 행동 지침입니다. 프로젝트별 규칙과 합쳐서 사용하세요.

**주의:** 이 지침은 속도보다 신중함을 우선시합니다. 간단한 작업은 상황에 맞게 판단하세요.

## 1. 코딩 전에 먼저 생각하기

**가정하지 말 것. 혼란을 숨기지 말 것. 절충점을 드러낼 것.**

구현 전에:
- 가정한 것을 명확히 밝혀라. 불확실하면 물어봐라.
- 여러 해석이 가능하면 모두 제시해라 — 조용히 하나만 고르지 마라.
- 더 간단한 방법이 있으면 말해라. 필요하면 반박해라.
- 뭔가 불명확하면 멈춰라. 무엇이 혼란스러운지 짚고 물어봐라.

## 2. 단순함 우선

**문제를 해결하는 최소한의 코드만. 추측성 코드는 금물.**

- 요청받지 않은 기능은 추가하지 마라.
- 한 번만 쓰이는 코드에 추상화를 적용하지 마라.
- 요청하지 않은 "유연성"이나 "설정 가능성"은 넣지 마라.
- 발생할 수 없는 시나리오에 대한 오류 처리는 하지 마라.
- 200줄로 짠 코드가 50줄로 될 수 있다면, 다시 써라.

스스로에게 물어봐라: "시니어 엔지니어가 보면 과하다고 할까?" 그렇다면 단순화해라.

## 3. 외과적 변경

**필요한 것만 건드려라. 네가 만든 것만 정리해라.**

기존 코드를 수정할 때:
- 인접한 코드, 주석, 서식을 "개선"하지 마라.
- 문제없이 작동하는 것을 리팩토링하지 마라.
- 기존 스타일이 마음에 들지 않더라도 맞춰라.
- 관련 없는 죽은 코드를 발견하면 언급만 해라 — 삭제하지 마라.

네 변경이 고아를 만들었을 때:
- 네 변경으로 인해 사용되지 않게 된 import/변수/함수는 제거해라.
- 요청받지 않은 이상 기존의 죽은 코드는 건드리지 마라.

기준: 변경된 모든 줄은 사용자의 요청으로 직접 추적 가능해야 한다.

## 4. 목표 중심 실행

**성공 기준을 정해라. 검증될 때까지 반복해라.**

작업을 검증 가능한 목표로 전환해라:
- "유효성 검사 추가" → "잘못된 입력에 대한 테스트를 먼저 작성하고, 통과시켜라"
- "버그 수정" → "버그를 재현하는 테스트를 작성하고, 통과시켜라"
- "X 리팩토링" → "전후로 테스트가 모두 통과하는지 확인해라"

여러 단계의 작업이라면 간략한 계획을 먼저 밝혀라:
```
1. [단계] → 검증: [확인 방법]
2. [단계] → 검증: [확인 방법]
3. [단계] → 검증: [확인 방법]
```

구체적인 성공 기준이 있으면 독립적으로 반복할 수 있다. 모호한 기준("되게 만들어")은 끊임없는 재확인을 낳는다.

## 프로젝트: EXIT 2026

### 개요

현대판 모세 이야기를 크라임씬 형태로 체험하는 이벤트용 웹 애플리케이션.
참가자들은 "EGYPT CITY 공사장 살인사건"을 수사하며, 최종 엔딩에서 용의자 C(M)의 정체가 모세임이 밝혀진다.

- 인원: 80명 / 조: 8개 / 시간: 120분
- 핵심 반전: 참가자는 모세 이야기를 체험하고 있다는 사실을 모름

### 기술 스택

- Framework: Next.js (TypeScript)
- Hosting: Vercel

### 주요 기능

| 기능 | 설명 |
|------|------|
| QR 페이지 | QR01~QR08 각 장소별 증거 접근 |
| 증거 수집 | 증거 10종 수집 및 보관함 표시 |
| 용의자 화면 | A(노동자 대표), B(현장 소장), C(회장 아들 M) 카드 |
| 최종 투표 | 조별 범인 선택 |
| 엔딩 화면 | 반전 공개 |

### 데이터 구조

```
evidence: id, title, description
suspect:  id, name, motive
timeline: time, event
```

### 작업 진행 기록

**세션 시작 시 반드시 `progress.md`를 먼저 읽어라.** 현재 작업 상태와 맥락을 파악한 후 작업을 시작해라.

작업을 시작하거나 완료할 때마다 `progress.md`를 업데이트해라.
- 작업 시작 시: 무엇을 할 것인지 기록
- 작업 완료 시: 무엇을 했는지, 변경된 파일 기록

새로운 기능을 구현할 때마다 `docs/01_md/EDIT_GUIDE.md`를 업데이트해라.
- 운영자가 직접 수정할 수 있는 기능이라면 반드시 추가
- 어느 파일의 어느 부분을 수정하면 되는지, 코드 예시와 함께 작성
